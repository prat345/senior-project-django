About

Part of Final Project, Development of Incident Database System of Autonomous Vehicle Testing

The Faculty of Engineering of Chulalongkorn University is aware of the future mobility trends and has been researching on the development of autonomous vehicles in campus. As the project is still in the development stages, the vehicles still need to be tested in various aspects such as navigation,ride comfort and incident scenario.

At present, the research team had to manually identify incidents and extract relevant information for further study which is both labor-intensive and time-consuming. Thus, this project proposes the implementation of an algorithm that identifies and extracts incident data from recorded test drives which will help increase efficiency while reducing manual workload. With the incident data, the research teams will be able to make more improvements and accelerate the development process. Ultimately moving towards safer, efficient, and reliable transportation for the community.


    Applications
  • Provide an interactive dashboard summarizing test drive data from database
  • Query and display test drive data including vehicle parameters and incident occurences
  • Query Incidents from labels which are based on their causes, display video from google drive, allow *RUD operations on labels and provide pagination for long query results
  • Test log for recording additional information about incident which developers can refer to when labeling incident. Allow CRUD operations and pagination
    Tools
  • Database: MongoDB Atlas (PyMongo&Djongo)
  • Framework: Django
  • Visualizations: Ploty
